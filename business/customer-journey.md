# Customer Journey Map - Project Shield

## Aware

### Trigger Event
- Open source project maintainers and contributors encounter scope creep issues, leading to project delays, dissatisfaction, and decreased productivity.

### Friction Points
- Lack of clear project scope definition and management tools.
- Time-consuming manual processes for defining and managing project scope.
- Miscommunication and disagreements among project members about project goals and priorities.

### User Emotions
- Frustration and stress due to project delays and dissatisfaction.
- Disappointment with the current state of project management tools.
- Desire for a solution that simplifies project scope management and prevents scope creep.

### Opportunities to Delight
- Providing a user-friendly, intuitive interface for defining and managing project scope.
- Offering real-time collaboration and communication features to facilitate project discussions.
- Integrating with popular open source project management platforms for seamless adoption.

### Metric
- Awareness level among open source project maintainers and contributors (e.g., website visits, social media engagement, newsletter sign-ups).

## Consider

### Trigger Event
- Open source project maintainers and contributors research potential solutions to manage project scope and prevent scope creep.

### Friction Points
- Evaluating multiple tools and comparing their features, pricing, and ease of use.
- Understanding the benefits and drawbacks of each tool for their specific project needs.

### User Emotions
- Skepticism about the effectiveness of new tools.
- Excitement about the potential benefits of a dedicated project scope management tool.
- Anxiety about the time and resources required to implement a new tool.

### Opportunities to Delight
- Offering a free trial or demo to allow users to experience the tool before committing.
- Providing clear and concise information about the tool's features, benefits, and pricing.
- Offering customer support and resources to help users make an informed decision.

### Metric
- Number of trial sign-ups or demos requested.
- Time spent on the website or engaging with marketing materials.

## Try

### Trigger Event
- Open source project maintainers and contributors decide to try Project Shield.

### Friction Points
- Learning the tool's interface and features.
- Integrating the tool with existing project management workflows.

### User Emotions
- Curiosity about the tool's capabilities.
- Optimism about the potential benefits for their project.
- Apprehension about the learning curve and potential disruptions to the project.

### Opportunities to Delight
- Providing comprehensive documentation and tutorials.
- Offering responsive customer support to address any issues or questions.
- Continuously updating the tool based on user feedback to improve the user experience.

### Metric
- Number of active trial users.
- User engagement metrics (e.g., number of projects created, number of tasks managed, number of discussions initiated).

## Adopt

### Trigger Event
- Open source project maintainers and contributors decide to adopt Project Shield for their project management needs.

### Friction Points
- Completing the onboarding process and setting up the tool for their project.
- Convincing other project members to adopt the tool.

### User Emotions
- Relief at finding a solution to manage project scope effectively.
- Satisfaction with the tool's features and ease of use.
- Pride in their decision to adopt a cutting-edge project management tool.

### Opportunities to Delight
- Offering a seamless onboarding process with personalized support.
- Providing incentives for early adopters (e.g., discounts, priority support).
- Encouraging user feedback and incorporating it into the tool's development.

### Metric
- Number of paid subscriptions.
- User retention rate (percentage of users who continue to use the tool after the trial period).

### Trigger Event
- Open source project maintainers and contributors expand their use of Project Shield to manage additional projects or features.

### Friction Points
- Learning new features or functionalities as the tool evolves.
- Managing multiple projects within the tool.

### User Emotions
- Excitement about the tool's growth and development.
- Satisfaction with the tool's ability to scale with their project needs.
- Confidence in the tool's long-term potential.

### Opportunities to Delight
- Offering regular updates and new features based on user feedback.
- Providing scalable pricing plans to accommodate growing project needs.
- Encouraging user engagement and collaboration within the Project Shield community.

### Metric
- Revenue growth from expanded usage.
- User engagement metrics (e.g., number of projects managed, number of tasks completed, number of discussions initiated).
